# Abstract

The Hospital Management System (HMS) is a comprehensive, full-stack web application designed to streamline and automate the administrative and clinical workflows of Al-Hayat Hospital. Built using modern technologies including ASP.NET Core 8.0 for the backend API and React.js for the frontend interface, the system provides an integrated platform for managing patients, doctors, appointments, medical records, prescriptions, billing, medicines inventory, departments, and rooms.

The system addresses the challenges of manual hospital management by providing:
- **Centralized Data Management**: All hospital data is stored in a secure SQL Server database with proper relationships and constraints.
- **Role-Based Access Control**: Different user roles (Admin, Doctor, Patient) have appropriate access levels.
- **Bilingual Support**: Full Arabic and English language support with RTL (Right-to-Left) layout for Arabic.
- **Real-time Dashboard**: Comprehensive statistics and analytics for hospital administration.
- **RESTful API**: Well-documented API endpoints for integration with mobile applications and third-party systems.

The HMS significantly reduces paperwork, minimizes errors, improves patient care quality, and enhances operational efficiency across all hospital departments.

# Introduction

## 1.1 Background

Healthcare institutions face increasing challenges in managing large volumes of patient data, coordinating between departments, scheduling appointments, maintaining medical records, and handling billing operations. Traditional paper-based systems are error-prone, time-consuming, and difficult to maintain. The need for a digital solution that can handle these complexities while ensuring data security and accessibility has become paramount.

## 1.2 Problem Statement

Al-Hayat Hospital requires a modern, integrated system to:
- Eliminate manual record-keeping and reduce human errors
- Provide quick access to patient medical history
- Streamline appointment scheduling and doctor availability management
- Automate billing and payment tracking
- Manage medicine inventory with expiry tracking
- Generate reports and analytics for decision-making
- Support both Arabic and English interfaces

## 1.3 Project Objectives

1. **Develop a centralized hospital management platform** that integrates all departments and services.
2. **Implement secure authentication and authorization** to protect sensitive medical data.
3. **Create an intuitive, bilingual user interface** accessible to staff with varying technical skills.
4. **Provide real-time dashboards and reporting** for administrative oversight.
5. **Build a scalable architecture** that can accommodate future growth and feature additions.
6. **Ensure data integrity and reliability** through proper database design and validation.

## 1.4 Scope

The system covers the following modules:
- **Patient Management**: Registration, profile management, medical history
- **Doctor Management**: Registration, specializations, schedules
- **Appointment Management**: Booking, status tracking, cancellation
- **Medical Records**: Diagnosis, treatment plans, vital signs
- **Prescription Management**: Creating and dispensing prescriptions
- **Billing System**: Invoice generation, payment tracking, insurance
- **Medicine Inventory**: Stock management, expiry alerts
- **Department & Room Management**: Resource allocation
- **Dashboard & Reporting**: Statistics and analytics
